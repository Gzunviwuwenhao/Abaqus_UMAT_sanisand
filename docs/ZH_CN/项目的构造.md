### sanisand本构模型的使用指南

Sanisand边界面模型是由Dafalias和Manzari在2004年首次提出, 是一个岩土工程领域广泛运用的砂土本构模型。我们可以在以下网址中下载原始的Sanisand本构模型的(Sanisand的代码链接: [umat-plaxis-SANISAND-CU](https://soilmodels.com/sanisand/).)
作者在学习该模型中发现, 原始的代码语法过于落后, 不便于阅读和二次开发。因此, 作者对原始代码进行了重构和优化, 使其更易于理解和使用。

#### 1. 构建工具包

##### 1.1 基础配置模块

首先我们先构建了一个Base_config的模块,用于存储全局变量和类型参数,这样在后续的开发中可以直接调用这些参数, 避免重复定义和硬编码。Base_config模块的代码如下:

```fortran
MODULE Base_config
  use, intrinsic :: iso_fortran_env, only: real32, real64, int32, int64, int8
  IMPLICIT NONE
  INTEGER, PARAMETER :: DP = KIND(1.0D0) ! 双精度参数
  INTEGER, PARAMETER :: SP = KIND(1.0E0) ! 单精度参数
  !
  real(DP), parameter :: PA = 101.325_DP   !< Atmospheric pressure (kPa)
  real(DP), parameter :: PI = 3.1415927_DP !< Pi constant 
  !> Kronecker delta tensor (identity matrix for 3D)
  real(DP), parameter :: DELTA(3, 3) = reshape( &
                         [1.0_DP, 0.0_DP, 0.0_DP, &
                          0.0_DP, 1.0_DP, 0.0_DP, &
                          0.0_DP, 0.0_DP, 1.0_DP],[3, 3]) !< 3x3 identity matrix
END MODULE Base_config
```


##### 1.2 材料参数配置模块

接下来我们继续构建一个Material_config模块, 用于存储Sanisand模型所需的材料参数。这样我们可以在主程序中直接调用这些参数, 使代码更加简洁和易读。为了区分这样材料参数与全局变量的区别，我们使用了一个type来封装所以的材料变量，这样在使用的时候不容易将材料参数与其他变量混淆。Material_config模块的代码如下:

```fortran
module Material_config
  use Base_config
  implicit none
  public :: PARAM
  private
  type param_
    real(DP) :: G0, NU
    real(DP) :: C
    real(DP) :: KH
    real(DP) :: VOIDC, KSI, LAC, MCS, EA
    real(DP) :: D1, DM
    real(DP) :: CH, NKP
    real(DP) :: FEVR, F0
  endtype param_
  ! initial type
  type(param_), parameter :: PARAM = param_(G0=125.0_DP, NU=0.1_DP, C=0.75_DP, &
                                            KH=0.03_DP, KSI=0.7_DP, LAC=0.05_DP, &
                                            MCS=1.25_DP, EA=0.1_DP, VOIDC=0.934_DP, &
                                            D1=0.1_DP, DM=3.5_DP, &
                                            CH=0.9_DP, NKP=3.0_DP, FEVR=5.7_DP, F0=0.45_DP)
contains

endmodule Material_config
```
